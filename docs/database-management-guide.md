# 数据库管理功能使用指南

## 概述

Hub Server 提供了完整的数据库管理功能，允许管理员通过 Web 界面监控、优化和清理数据库。

## 功能特性

### 1. 数据库统计

实时显示数据库状态：
- **数据库大小**: 当前数据库文件大小（MB）
- **总记录数**: 所有表的记录总数
- **数据表数量**: 数据库中的表数量
- **各表详情**: 每个表的记录数和占用空间

### 2. 数据库优化

执行以下优化操作：
- **ANALYZE**: 更新查询优化器统计信息，提升查询性能
- **VACUUM**: 回收已删除数据占用的空间，减小数据库文件大小
- **索引优化**: 重建索引以提高查询效率

**建议频率**: 每月执行一次，或在大量删除数据后执行

**注意事项**:
- 优化过程可能需要几分钟时间
- 优化期间数据库性能可能暂时下降
- 建议在低峰期执行

### 3. 数据归档

自动删除旧数据以释放空间：

**配置选项**:
- **浏览记录保留**: 保留最近 N 个月的浏览记录（默认 6 个月）
- **下载记录保留**: 保留最近 N 年的下载记录（默认 1 年）
- **同步历史保留**: 保留最近 N 个月的同步历史（默认 3 个月）

**警告**:
- 归档操作会永久删除数据，无法恢复
- 执行前会显示确认对话框
- 建议先备份数据库

## 使用方法

### 访问数据库管理

1. 以管理员身份登录 Hub Server
2. 进入"管理后台"页面
3. 点击"数据库管理"标签

### 查看统计信息

进入数据库管理标签后，系统会自动加载统计信息，显示：
- 数据库总体状态（大小、记录数、表数量）
- 各数据表的详细统计（记录数、占用空间）

### 执行优化

1. 在"数据库优化"区域，点击"立即优化"按钮
2. 确认优化操作
3. 等待优化完成（通常需要 1-5 分钟）
4. 查看优化后的统计信息

### 执行归档

1. 在"数据归档"区域，配置保留时间：
   - 调整浏览记录保留月数（1-24 个月）
   - 调整下载记录保留年数（1-5 年）
   - 调整同步历史保留月数（1-12 个月）

2. 点击"执行归档"按钮

3. 确认删除操作（会显示将要删除的数据范围）

4. 等待归档完成

5. 查看删除的记录数和更新后的统计信息

## API 接口

### 获取数据库统计

```json
GET /api/admin/database/stats
Authorization: Bearer <token>
```

### 优化数据库

```json
POST /api/admin/database/optimize
Authorization: Bearer <token>
```

### 归档旧数据

```json
POST /api/admin/database/archive
Authorization: Bearer <token>
Content-Type: application/json
```
